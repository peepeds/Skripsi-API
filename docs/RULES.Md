# Clean Code Rules

1. Use **clear and descriptive names**  
2. Avoid **uncommon abbreviations**  
3. Follow **Single Responsibility Principle (SRP)**  
4. Keep **functions small and focused**  
5. Avoid **code duplication (DRY)**  
6. Use **constants instead of magic numbers**  
7. Write **readable code, not just working code**  
8. Maintain **consistent structure and style**  
9. Avoid **excessive comments**  
10. Write comments that explain **“why”, not “what”**  
11. Use **early returns** to reduce nesting  
12. Limit **function parameters (ideally ≤ 3)**  
13. Apply **separation of concerns**  
14. Use **clear error handling**  
15. Remove **dead/unused code**  
16. Avoid **hidden side effects**  
17. Use **appropriate data types**  
18. Write **testable code**  
19. Be consistent with **coding conventions**  
20. Refactor regularly
